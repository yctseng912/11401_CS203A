# 堆疊 (Stack) & 隊列 (Queue) 

## 一、 堆疊 (Stack)

堆疊是一種遵循 **後進先出 (LIFO, Last-In-First-Out)** 原則的線性資料結構。

### 1. 基本特性

* 元素只能從稱為「頂端 (Top)」的一端進行添加或移除。
* 最後加入堆疊的元素會是第一個被移除的。

### 2. 核心操作 (ADT Operations)

| 操作 | 說明 | 複雜度 |
| --- | --- | --- |
| **Push(x)** | 將元素 x 加入堆疊頂端 | O(1) |
| **Pop()** | 移除並回傳堆疊頂端的元素 | O(1) |
| **Peek/Top()** | 回傳頂端元素但不移除它 | O(1) |
| **IsEmpty()** | 檢查堆疊是否為空 | O(1) |

### 3. 實作方式

* **陣列實作 (Array)**：使用固定大小的陣列，並以一個變數（通常是 `top`）紀錄索引位置。
* **鏈結串列實作 (Linked List)**：動態分配記憶體，堆疊頂端即為串列的頭節點 (Head)。

---

## 二、 隊列 (Queue)

隊列是一種遵循 **先進先出 (FIFO, First-In-First-Out)** 原則的線性資料結構。

### 1. 基本特性

* 元素從「後端 (Rear/Back)」加入，並從「前端 (Front)」移除。
* 第一個加入隊列的元素會是第一個被移除的。

### 2. 核心操作 (ADT Operations)

| 操作 | 說明 | 複雜度 |
| --- | --- | --- |
| **Enqueue(x)** | 將元素 x 加入隊列後端 | O(1) |
| **Dequeue()** | 從隊列前端移除並回傳元素 | O(1) |
| **Front()** | 讀取隊列前端元素但不移除 | O(1) |
| **IsEmpty()** | 檢查隊列是否為空 | O(1) |

### 3. 實作類型

* **普通隊列 (Linear Queue)**：使用陣列實作時，可能會遇到「假溢滿」問題。
* **環狀隊列 (Circular Queue)**：將陣列首尾相連，有效利用被釋放的空間。
* **雙端隊列 (Deque)**：允許在前端與後端同時進行插入與刪除。

---

## 三、 堆疊與隊列的比較

| 特性 | 堆疊 (Stack) | 隊列 (Queue) |
| --- | --- | --- |
| **原則** | LIFO (後進先出) | FIFO (先進先出) |
| **存取點** | 僅一端 (Top) | 兩端 (Front, Rear) |
| **主要操作** | Push / Pop | Enqueue / Dequeue |
| **應用場景** | 遞迴調用、撤銷 (Undo) | 任務排程、資源共享 |

---

## 四、 實際應用範例

* **Stack 應用**：
  * **算術表達式轉換**：例如將中序表示法 (Infix) 轉換為後序表示法 (Postfix)。
  * **函數調用管理**：作業系統使用堆疊來追蹤函數的調用路徑。
  * Undo & Redo

* **隊列應用**：
  * **生產者-消費者模型**：在併發編程中平衡不同速度的任務處理。
  * **緩衝區管理**：例如印表機的作業列或網絡數據包傳輸。
